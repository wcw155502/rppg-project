# Issue tracker: GitHub

Issues and PRDs for this repository live in [GitHub Issues](https://github.com/wcw155502/rppg-project/issues).

## Repository

- Owner/repository: `wcw155502/rppg-project`
- Issue tracker: GitHub Issues
- Pull requests as a triage request surface: no

## Conventions

- Create, read, comment on, label, and close work items through GitHub Issues.
- When GitHub CLI is installed, use `gh issue` commands from a clone with `origin` configured.
- Until GitHub CLI is installed, use the GitHub web interface for issue operations.
- When an engineering skill says to publish work to the issue tracker, create a GitHub issue.
- When an engineering skill says to fetch a relevant ticket, read the corresponding GitHub issue and comments.

## Pull requests

External pull requests are not treated as a triage request surface by default.
